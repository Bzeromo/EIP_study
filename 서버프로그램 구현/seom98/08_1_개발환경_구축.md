# 1. 개발 환경 구축

## 1-1. 개발 환경 구축의 개념

-   **개발 환경**: 소프트웨어 개발에 필요한 도구와 환경을 설정하는 과정
-   개발자 생산성 향상과 일관된 개발 환경 제공

## 1-2. 개발 도구의 분류

### 빌드 도구:

-   소스 코드를 실행 가능한 프로그램으로 변환하는 도구
-   의존성 관리 및 프로젝트 빌드 자동화

### 구현 도구:

-   코드 작성과 편집을 위한 통합 개발 환경
-   디버깅 및 코드 분석 기능 제공

### 테스트 도구:

-   소프트웨어 품질 검증을 위한 도구
-   단위 테스트, 통합 테스트, 자동화 테스트 지원

### 형상 관리 도구:

-   소스 코드 버전 관리 및 변경 이력 추적
-   팀 협업을 위한 코드 공유 및 병합 기능

## 1-3. 개발 환경 구성요소

### 하드워어 개발 환경

-   서버 하드워어: 웹서버, 웹 애플리케이션 서버, 데이터베이스 서버, 파일 서버
-   클라이언트 하드웨어: 클라이언트 프로그램, 웹 브라우저, 모바일 앱, 모바일 웹
-   소프트웨어: 운영체제, 미들웨어, DBMS

### 형상 관리

#### 형상 관리의 개념

-   소프트웨어 개발 과정에서 발생하는 모든 변경사항을 체계적으로 관리하는 활동

#### 형상 관리의 목적

-   변경사항 추적 및 이력 관리
-   버전 간 일관성 유지
-   팀 협업 효율성 증대

#### 형상 관리의 절차

-   형상 식별: 관리 대상 항목을 식별하고 분류
-   형상 통제: 변경 요청에 대한 검토 및 승인 절차
-   형상 감사: 형상 항목의 무결성 및 일관성 검증
-   형상 기록: 모든 변경사항을 문서화하고 기록

#### 소프트웨어 형상 관리 도구 유형

-   공유폴더 방식: 네트워크 공유 폴더를 통한 파일 공유
-   클라이언트/서버 방식: 중앙 서버에서 버전 관리
-   분산 저장소 방식: 각 개발자가 전체 저장소를 복제하여 관리

#### 소프트워어 형상 관리 도구별 특징

-   RCS: 단일 파일 단위의 버전 관리
-   CVS: 클라이언트/서버 기반의 버전 관리
-   SVN: 중앙집중식 버전 관리 시스템
-   GIT: 분산 버전 관리 시스템

### 백업

#### 백업 개념

-   데이터 손실 방지를 위해 중요한 정보를 복사하여 보관하는 작업

#### 백업 유형

-   전체 백업: 모든 데이터를 완전히 복사
-   차등 백업: 전체 백업 이후 변경된 모든 파일을 백업
-   증분 백업: 마지막 백업 이후 변경된 파일만 백업
